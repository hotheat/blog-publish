安装步骤

初次运行 Git

git config --global user.email "Your email "

git config --global user.name "Your name"

Hexo 常用命令

安装 Hexo 及相关组件

安装 Hexo

新建 Blog 目录，必须是空目录，否则 hexo init 报错。

npm install -g hexo-cli

生成 Hexo 项目

本地调试运行

此时，在浏览器输入「localhost:4000」可以看到 Hello world 页面。

目录结构

  1. scaffolds 目录

    新建页面的模板，执行新建命令时，是根据这里的模板页来完成的。

  2. 主要配置文件

    _config.yml 站点配置文件

    themes/*/_config.yml 主题配置文件

安装 next 主题

  1. next 官方网站上下载

    git clone https://github.com/iissnan/hexo-theme-next themes/next

    新版本的 NexT 6.x 还对部分功能的使用做了优化。把很多功能比如字数统计，顶部进度条等放到了主分支里。具体可以参考Hexo NexT 主题6.x版本的使用配置与美化。

  2. 修改站点配置文件中的 themes 字段为 next

  3. 使用组合命令运行

    hexo s -g

添加文章

  1. 添加文章

    hexo new "文章名"

  2. 添加内容

    在 blog/source/_post 文件夹下有文章名.md文件。

    将要写的内容添加到 --- 下

  3. 本地提交

    hexo s -g

    刷新浏览器可以看到刚写的文章了

利用 Github Pages 实现远程访问

新建仓库

  1. 在 github 中创建新仓库，仓库名字格式「username.github.io」，勾选用初始化 README

  2. 在 Setting 页面的 Github Pages 中，Source 显示为 master branch

绑定 SSH Key

  1. 在本地找到 id_rsa.pub 文件，进入 github，Settings $\rightarrow$ SSH and GPG keys $\rightarrow$ New SSH key，将 ssh key 粘贴进去
  2. ssh -T git@github.com验证 ssh key 是否添加成功

部署网站

  1. 修改 Blog 目录下站点配置文件，repository 改为自己的仓库名

    • repository 使用 ssh 方式进行复制
    • 每个冒号后面都有空格

  2. 执行命令

    在浏览器中输入「hotheat.github.io」即可访问

有时本地文件会出错，执行 hexo clean 即可清除 public 文件夹，再重新提交

绑定域名

  1. 注册域名，推荐 godaddy.com 和「万网」，开始时注册的腾讯云域名，在同时绑定 coding.net 后子域名负载均衡超限需要花钱

  2. 添加主机记录 www， 记录类型 CNAME，记录值仓库名：hotheat.github.io，保存

  3. 再添加主机记录 @， 记录类型 CNAME，记录值仓库名：hotheat.github.io，保存

  4. Github setting 里 Github page 页面中填入新的域名。保存后，此时仓库中会多出 CNAME 文件，里面是域名信息

  5. 下次执行 hexo d 时，CNAME 文件会被删掉。需要本地在 source 目录中新建 CNAME 文件，里面写上域名地址 「www.hotheat.top」，下次执行 hexo d 时会自动复制到 post 文件夹下

同时部署到 Coding.net

因为 Github 屏蔽了百度的爬虫，国内流量的解析到 Coding，国外的解析到 Github。Coding.net 在国内的访问速度也比较快。

注册 Coding.net $\rightarrow$ 绑定 ssh 证书 $\rightarrow$ 站点配置文件中配置 deploy 方式（添加内容如下） $\rightarrow$ hexo d -g 推送 $\rightarrow$ 绑定域名（主机记录同样填写 www 和 @）

  1. Coding.net 绑定腾讯云后免费升级到银牌会员，可以像 Github Page 一样对自定义域名进行绑定了

  2. 在 Coding Pages 里申请 SSL 证书时，会同时解析国外线路和国内线路，当解析到国外线路时，申请证书无法通过，需要将 github.io 线路暂停，待申请通过后再启用。DNS 有缓存，修改后约有 5 分钟生效时间。在域名解析时尤其不能心急

  3. 不需要选中「强制 HTTPS 访问」

  4. coding.me 跳转页的广告删除请参考文章 http://stevenshi.me/2017/12/22/hexo-codingNet/。再点击头像右下角，切换旧版，勾选 已放置 Hosted By Coding Pages 提交审核。

分类和标签

分类

  1. 执行命令

    hexo new page categories

  2. 在提示生成的 index.md 中添加 type: “categories”

  3. 在要添加分类的文章 Header 中添加 categories 属性值。一篇文章只能属于一个分类，如果在下一行添加 -xxx，那么 -xxx 将属于上一个分类的子类

标签

  1. 标签和分类同样的设置，执行 hexo new page tags

  2. 在 index.md 中添加 type: “tags”

  3. 在要添加分类的文章 Header 中添加 tags 属性值。与 categories 不同，一篇文章可以有多个 tags

新建文章时自动生成 categories

在 scaffold/post.md 中的 tags: 上面加入 categories:，在 hexo new "文章名"时，就可以自动添加 categories 项。

关于（About）

    hexo new page about

出现问题

  1. 点击 使用 GitHub 登录后跳转到主页

    进入控制台发现报错 [VM646:1 GET https://api.github.com/user **401** (Unauthorized)]，怀疑是开始注册时填写的回调网站 hotheat.github.io有问题。

    解决：将注册时的 callback URL改成 https://www.hotheat.top，在主题配置文件中修改相应的 ClientID 和 ClientSecret，而 repo 不用改，就可以点击登录评论。

  2. Chrome 中点击评论按钮捕捉不到 mousedown 事件，Firefox 则可以正常添加评论

    报错信息：

    解决：最终测试发现，是 Chrome 浏览器中 Tampermonkey（油猴）插件中「网页限制解除」脚本导致的。debug 过程满满地都是泪啊~
