已经讲过的三大块重点，用计算图表示任何一个函数、两层神经网络和卷积神经网络。

卷积神经网络能够保持空间的输入结构。每个卷积核作为滤波器在空间上滑动，多个滤波器构成特定深度的输出。

需要的是学习这些所有的权重和系数。用 Mini-batch SGD 的方法。正向传播，loss，反向传播，梯度更新。

这节课讲的细节：

  1. 如何建立起神经网络
  2. 选择激活函数，
  3. 数据预处理，权重初始化，正则化，梯度检查
  4. 训练中的动态变化，如何监控（Babysitting），如何选择参数的特定更新规则，怎样做超参数优化
  5. 模型评估和模型集成

激活函数

激活函数至关重要，如果去掉，那么两个线性层会合二为一，重新变成一个线性函数。

sigmoid

每个元素都在 [0, 1] 范围内，中间区域可以看作线性函数。可以看作神经元的饱和放电率。

若干问题：

  1. 饱和神经元将使梯度消失。当输入值过大或过小（接近0或1）时会饱和，梯度接近 0，局部梯度非常小，乘以返回的上游梯度将得到一个很小的梯度，零梯度将阻止梯度传递。在权重初始化时需要特别留意，如果权重初始化过大，大多数神经元将饱和。

  2. 输出是非零中心的函数，假设对于 $wx + b$，若输入 x 一直为正，上游传回的梯度为正数或负数，对 w 求导后局部梯度为 x，乘以局部梯度后，w 的梯度将一直得到全是正数或负数，相当于沿上游梯度的正负号进行传递，总是沿同一个方向移动。

    对于梯度更新来说非常低效，以下图为例，第一象限和第三象限是全为正或全为负，蓝色箭头代表 w 的梯度方向，我们不能沿 w 的方向更新参数，而是只能沿红色箭头中表示的两个方向去更新梯度。出现z字型下降

    如果使用均值为 0 的数据，将得到正和负的值，就不会陷入上述梯度更新的问题。

  3. 指数函数的计算代价有些高，但不需要太注意这点。

tanh

和 sigmoid 很类似，但是是以 0 为中心的，不会有上述第二个问题，但仍会有梯度消失的问题。

可以看成是一个简单放大的 sigmoid 神经元，

Relu

$f(x) = max(0, x)$

在元素上进行操作。

  1. 不会在正数区域产生饱和现象，输入空间的一半都不会有饱和
  2. 计算速度快
  3. 收敛速度快，比 sigmoid 和 tanh 快约 6 倍

不是以 0 为中心，负半轴上梯度会消失。在 x=0 处导数不存在。

负半轴上称为 dead ReLU，不会被激活和更新，可能的原因是权重初始化非常差；学习率太高，权值不断波动，被数据的多样性淘汰，通常在训练开始时较好，某个时间点后挂掉；大多数使用 ReLU 的网络会有部分的节点挂掉，通常还会再去检查。

通过合理的设置学习率，可以是这种情况的概率降低。

其他 ReLU

Leaky ReLU：$f(x) = max(0.01x, x)$负轴上有一个微小的斜率，仍然收敛快，而且不会出现梯度消失。

PReLU：$f(x) = max(\alpha x, x)$，倾斜区间斜率是一个超参数$\alpha$，可以进行学习。输出是以 0 为中心的。

ELU，介于 ReLU 和 Leaky ReLU 之间，输出接近零值中心，但是在负区间没有倾斜，具有负饱和机制，对噪声有更强的鲁棒性。

Maxout “Neuron”

$max(w_1^Tx + b_1, w_2^T + b_2)$，参数数量由于 $w_1$ 和 $ｗ_2$ 会翻倍，具有ReLU单元的优点（线性操作和不饱和），但是梯度不会消失，也不会饱和。ReLU 和 Leaky ReLU 是 Maxout 的特殊情况，ReLU 是 $w_1, b_1=0$ 时。

实际使用中，在同一网络中混用不同类型的神经元很少见，ReLU 是最常用的，设置好学习率，监控网络中死亡的神经元的比例，当有单元死亡问题困扰时，可以尝试 ReLU 的变体，但是不要使用 sigmoid。

神经网络相关

当我们说 N 层神经网络时，没有把输入层算入。单层神经网络就是没有隐层的。

输出层（最后一层）一般不会有激活函数的，因为最后的输出层大多用来表示分类评分值或者回归的实数值。

神经网络的尺寸一般指神经元个数和参数个数，上述第二个网络的神经元个数是 $4+4+1=9$，参数为 $3\times4 + 4\times 4+4\times1=32$ 个权重，加 $4+4+1=9$ 个偏置，共41个学习的参数。

神经网络的表达能力

对于任意连续函数$f(x)$，都存在至少含1个隐层的神经网络$g(x)$（具有合理的激活函数），使得任意x，均有$|f(x)-g(x)|<\epsilon$。也就是说，神经网络可以近似任意连续函数。

神经网络的层数

在实践中，3层的网络比2层的表现好，继续加深（比如4、5、6层后）很少有太大的帮助。卷积网络不同，在卷积网络中，深度是一个很重要的因素（以10为数量级），一个解释是图像拥有层次化的结构，比如脸由眼睛构成，眼睛又由边缘构成。

神经元个数

上图表示只有一个隐层的，不同神经元个数的神经网络，更多的神经元可以分类更复杂的数据，但是可能对训练数据造成过拟合。对噪声有很强的拟合能力，没有重视数据间的潜在基本关系。而神经元数量较少时，在测试数据中的泛化能力会更好。

在实践中，使用L2正则，dropout和输入噪音等控制过拟合的方法比减少神经元数目要好得多。

一般不使用减少神经元数目来控制过拟合。因为小网络很难使用梯度下降等算法进行训练。虽然小的神经网络的局部极小值更少，而且也容易收敛到这些局部极小值，但是，这些最小值一般很差，损失值很高，呈现出多变性，运气好时收敛到一个好地方，运气不好时，收敛到一个不好的极值，大网络的局部极小值表现更好，损失更小。

正则化强度是控制神经网络过拟合的好方法。随着正则化强度增加，决策边界变得更加光滑。

应该尽可能使用大网络（神经元数量多），使用正则化技巧来控制过拟合。

设置数据和模型

数据预处理

normalize 的作用：

如果 x 全为 0 或者有一些非零的数据，仍然会遇到前面讲过的梯度下降效率低，以及得到次最优的优化结果。

均值减法：对数据的每个独立特征减去均值，在每个维度上将数据中心迁移到原点，对于$N\times D$的数据，$X-=np.mean(X,axis=0)$。在图片训练中，一般只做零均值处理就行，减去整个图像的均值（AlexNet）或每个 channel 上减去 channel 的均值（VGGNet）。

机器学习中，要做最优化时，需要所有的特征在相同值域内，而且特征贡献相同，而图像中的像素已经确保得到了相对稳定的范围和分布，没必要归一化太多。对于 PCA 和白化也不会做太多。

归一化：一般在每个维度上除以标准差或者使每个维度的最大和最小值是1和-1。这个操作在确信不同的输入特征有不同的值域范围时才有意义。

零均值仅在第一层解决了 sigmoid 问题， 在后面的一些层中，这个问题可以还会更严重。

PCA 和白化

PCA 的一种解释是最大化投影方差，假设 $w$ 是投影方向，对于第 $i$ 个数据点 $xi$ 在 $w$ 上的投影可以用 $x_i \cdot w = x_i^T w$ 来表示，$X={x_1, x_2, …x_n}$，$x_i$ 是列向量。投影后的方差为 $var=\frac{1}{n} \Sigma{i=1}^{n}( x_i^T w-E(X^Tw))^2 = \frac{1}{n} (X^Tw-E(X^Tw))^T(X^Tw-E(X^Tw))$。

如果对于 $xi$ 中心化后，也就是 $u = \frac{1}{n}\Sigma{i=1}^nx_i;x_i=x_i - u$，

$E(X^Tw)= \frac{1}{n}\Sigma{i=1}^nx_i^Tw=(\frac{1}{n}\Sigma{i=1}^nx_i^T)w=0$，这也是进行中心化的意义。

$var = \frac{1}{n} (X^Tw)^T(X^Tw) = \frac{1}{n} (w^TXX^Tw)$，因为做了中心化，$XX^T$ 就表示数据 X 的协方差矩阵，协方差矩阵的第 $(i, j)$ 个元素代表第 i 个和第 j 个维度的协方差，协方差矩阵是对称且正定的。$X^T X$ 是正定阵。

S 是特征值的平方构成的对角阵。

PCA 降维：np.linalg.svd 返回的 U 是按特征值大小排列的，选择方差最大的前100维，丢弃那些没有方差或方差较小的数据，我们认为信号具有较大的方差，噪声具有较小的方差。

将已经去中心的数据投影到特征向量上，去除数据间的相关性，这一步可以理解成对数据 X 进行旋转，变换基向量为特征向量方向。旋转的操作将协方差矩阵变为对角阵。

白化：

输入数据是特征向量为基向量的数据，在各个方向上除以特征值（S 的开方），对每个维度进行归一化。几何解释是：协方差矩阵可以理解成空间中的椭圆或椭球体，主轴方向上的长度是协方差矩阵 $X^TX$ 的特征值的开方，根据 SVD 可以知道 $X^TX$ 的特征值为 $\sigma_1$，除以 S 的开方后，类似于将椭圆变成圆的操作，将协方差矩阵变为单位矩阵。

1e-5 为了防止分母为 0。可能在变换过程中夸大数据的噪声，可以用更强的平滑解决，（采用比 1e-5 更大的值）

实际上，在卷积网络中并不会采用这些变换，但是对数据零中心化还是非常重要的。

实践

先将数据分成训练、验证和测试集，然后从训练集中求图片平均值，从训练/验证/测试集中减去这个平均值，而不是针对所有的图片求平均，然后减去这个值。

权重初始化

如果权重初始化为 0，所有的神经元将会做同样的事情，但并不是死掉，每个神经元将输出相同的数值，相同的梯度，以相同的方式更新，失去了神经元不对称性的源头。而我们希望的是每个神经元都可以做不同的事情。

所有在初始化时，可以从一个概率分布中采样，小随机数初始化。

对于小型网络适用，但对于结构深的网络， 随着每次乘以一些较小的随机数，激活值会迅速减小，最后激活值将得到一堆 0。反向传播中，由于 $WX$ 对参数 w 求导后的局部梯度将得到 x，正向传播的 x 很小，所以局部梯度很小，所以本地梯度很小，所以参数几乎得不到更新。网络崩溃。

如果增大权重会怎样？网络饱和

权重变大，如果用 tanh 激活函数，将会变得饱和，反向传播中的梯度将一直为 0。

层数深的网络该如何初始化参数？

Xavier 初始化。

在上图中可以看出，如果每层采用零均值和标准差的高斯分布初始化参数，随着网络的深度增加，激活值会迅速减小，反向传播中的“梯度信号”下降，就会出现问题。也就是说，随机初始化的神经元在输出数据的分布的方差在增大。而我们希望得到的是每一层的输出方差尽可能相等。在除以输入数据的平方根后，是神经元的输出方差归一化到 1。实践证明这样做也可以提高收敛速度。

下面是证明过程：

权重 $w$ 与输入 $x$ 内积为 $s = \Sigma_{i}^{n}w_ix_i$，还没有进行激活函数，$s$ 的方差

$Var(s) = Var(\Sigma_i^nw_ix_i)$

根据方差的性质，可以得到

$Var(s) = \Sigma_i^nVar(w_ix_i) = \Sigma_i^n([E(w_i)]^2Var(x_i) + [E[(x_i)]^2Var(w_i) + Var(x_i)Var(w_i))$

有了 BN 后， $x_i$ 是零均值，$w_i$ 初始化时满足零均值。

$Var(s) = \Sigma_i^nVar(x_i)Var(w_i)$

输入 x 和权重 w 满足独立同分布

$Var(s) = (nVar(w))Var(x)$，要使输出方差 $Var(s)$ 等于 $Var(x)$，$Var(w) = \frac{1}{n}$

要使方差等于 $\frac{1}{n}$，原来的$w$ 满足均值为0，方差为 1 的标准高斯分布，所以每个 $w_i$ 乘以 $\sqrt{\frac{1}{n}}$ 后，方差变为 $\frac{1}{n}$，于是就有了 $W = np.random.randn(fan_in, fan_out)/np.sqrt(fan_in)$。

现在$w_i$ 的方差变成 $Var(w) = \frac{1}{fan_in}$，类似的，还有另外两种方差形式

只考虑输出形式，$Var(w) = \frac{1}{fac_out}$；

同时考虑输入和输出形式，$Var(w) = \frac{2}{fac_in+ fac_out }$，概率统计上均匀分布的方差为 $Var = \frac{(b-a)^2}{12}$，所以这种形式的初始化 w 服从以下均匀分布 $W\sim U[-\frac{\sqrt{6}}{\sqrt{fac\in+fac_out}}, \frac{\sqrt{6}}{\sqrt{fac\in+fac_out}}]$

实践中，针对 ReLU 神经元的特殊初始化，神经元的方差是 $\frac{2}{n}$。

$W = np.random.randn(fan_in, fan_out)/np.sqrt(2/fan_in)$

如果输入维度较小时，将得到一个较大的权重，以得到相同的输出方差；相反，输入维度较大时，得到的权重较小，来匹配输出方差。

激活函数是 tanh 的权值分布

激活函数是 ReLU 的权值分布，每次会消除一半的神经元，相当于将方差减半，高斯分布会逐渐收缩，神经元越来越多地失活。

可以通过除以 2 来解决这个问题。

一般的经验法则是从 Xavier 法开始。

Batch Normalization

目的是单位高斯激活，在训练的每一层都能使数据保持高斯分布。因为神经网络中涉及很多层的叠加，每一层的参数更新会导致上层的输入数据分布发生变化，使每个神经元的输入数据不再是“独立同分布”。同时，下层的输入变化可能趋向于变大或变小，使上层落入饱和区，学习过早停止。也就是说，层内操作作用使各层的输出与对应的输入信号分布不同，差异随着网络深度增大而增大，这一误差随后又在梯度反向传播中被进一步放大。但是最终的 label 是不变的，这种训练过程中出现的分布与理想分布发生偏差的现象被成为Internal Covariate Shift。Covariate Shift的概念与2000年在统计学领域被提出，论文作者将原始的端到端的Covariate Shift引入到网络的每一层，因此称为Internal Covariate Shift。

BN的目标也就是解决分布偏差问题

在 BN 之前多采用白化处理，但涉及协方差矩阵的计算，计算成本很高。

对训练过程中 Batch 样本的每个维度分别计算均值和方差，经过减去均值除以方差后得到每个维度上均值为 0，方差为 1 的分布，随后在进行仿射变换，防止网络的表征能力下降。通常是在全连接层或卷积层之后，激活函数之前插入。在卷积层中，每个激活图都有一个均值和方差。

BN 能够允许更高的学习率，减少了对初始化参数的依赖性。

为了使模型的表达能力不因规范化而下降，获得非线性的表达能力需要再将平移参数和缩放参数加回去，比如采用 tanh 非线性函数时，这样的归一化操作，将数据限定在激活函数的线性区域中，避免出现过饱和，但有一点点饱和也无所谓。需要额外的缩放和平移操作。$\gamma$ 和 $\beta$ 需要通过学习得到。

BN 的总结：

  1. 放宽了对网络初始化的要求。对不好的初始值有更好的鲁棒性。

    横坐标是 weight decay 值，范围为 $[10^{-4}, 10^{0}]$，可以看出 BN 有更好的稳定性。

  2. BN 也可以看作是正则化的方法。通过对给定的训练样本进行批采样，对这一批样的输入数据进行归一化，不再对给定的训练样本提供确定的值，类似于向 x 中加入一些噪声和抖动。在一些任务中可以省去 Dropout。

  3. 放宽了对 learning rate 的要求。

  4. 测试集上的 BN 是针对所有 minibatch 的指数加权平均作为这一隐藏层的平均值和标准差。

但是对于 BN 放在什么位置，在哪些层增加BN？是否需要 Dropout？要不要使用 Weight Decay？这些需要找到一个平衡点。

对于批量非常小时，应该如何处理？

BN 更多的用在标准卷积网络中，对于像强化学习这种批次样本非常小时，准确率可能会变小，可能要用更多的样本计算均值和方差。

卷积网络中通常希望保留一些空间结构，只找到训练样本的经验均值和方差。

BN 的过程并没有改变权重。

Babysitting the Learning Progress

如何观察训练过程以及在训练过程中调整参数？

  1. 数据预处理
  2. 确定网络结构
  3. 初始化网络，确定损失函数是合理的，比如 Softmax loss 中，随机初始化参数后，在不用正则化的前提下，最终的损失是 $-log(0.1)$。
  4. 加入正则项后，观察 loss 情况。loss 将会有上升。
  5. 训练过程。最好先从小数据集开始，小数据集上会拟合得非常好，损失时是否能够降为 0，训练集上的正确率是 100%。比如说选择 20 个样本，关闭正则化，使用 sgd 下降等。

使用相对误差

分母上取最大值或者取和。

对于不可导的点，相对误差 < 1e-4 就可以，如果是激活函数是连续函数（tanh或sigmoid），误差通常小于 1e-7。出现不可导的原因可能是ReLU、SVM损失、Maxout神经元引入等，数值梯度可能会突然计算出一个非零的梯度值。

网络深度越深，相对误差越高。

要确保用于比较的解析梯度和数值梯度不能过小，比如绝对值小于 1e-10，如果过小，可以暂时用常数将损失函数的数值扩展到更好的范围，比较理想的是浮点数的指数为 0 时的数量级。

梯度检查中使用越少的数据点，含有不可导点的几率越低；h 要谨慎设置，一般为 1e-4 或 1e-6；为防止参数随机初始化可能导致的不正常边界情况，掩盖了梯度没有正确实现的事实，一般在网络学习一段时间后，等损失函数开始下降后再进行梯度检查。

正则化损失可能会吞没掉数据损失，这种情况梯度主要来源于正则化部分。推荐先关掉正则化对数据损失单独检查，然后对正则化做检查，对正则化检查时，可以关掉数据损失或提高正则化强度。

做梯度检查时要关闭 Dropout 和 augmentation。

要确保所有不同的参数都抽取一部分来做梯度检查。

学习之前的合理性检查

  • 特定情况下的正确损失。比如不用正则化时，随机小参数初始化的 Softmax 的每个类别的损失为 $-log(0.1)$
  • 提高正则化强度后损失变大
  • 最重要的一步，不用正则化时，小数据集过拟合，损失为0。但可能由于某些错误可能对小数据过拟合，但整个数据集上没有任何泛化能力。

检查学习过程

上述完整性检查结束后，拿出所有的训练数据，加上小的正则项。从较小的学习率开始调。学习率是最重要的参数。

学习率太低时，loss 几乎不会下降，但是准确率却在上升，原因是虽然损失的分布仍然分散，但是所有的分布在朝着正确的方向移动，参数更新也在正确的方向上，准确率可能会发生突变。

学习率太大时，cost 为 nan 时，需要你降低学习率了。一般用的交叉验证的学习率的范围是[1e-3, 1e-5]。

学习率

学习超参数，在训练集上训练，在验证集上验证。

不同学习率下的 loss 曲线，一个epochs 代表每个样本数据都被观察过一次

过低的学习率导致算法的改善是线性的，过高的学习率可以让损失值快速下降，但最终停在一个不好的损失值上。因为学习率太高，引起随机震荡，不能停在较好的点上。

loss 曲线的震荡程度和 batch size 有关，batch size 越小，震荡越大。

训练集和验证集准确率

训练集和验证集上的准确度曲线之间的空隙代表了模型过拟合的程度。相差太大时，可能是发生过拟合了。过拟合时，应该增大正则化强度（更高的 L2 权重惩罚和更多的随机失活）或收集更多的数据或减少隐藏层数量。如果训练集和验证集曲线几乎相同，那么应该增大模型容量。

权重更新前后比例

通过比较更新前后参数的二范数的比例，通常比例在 1e-3 左右是可以的。当这个比值太小时，学习率可能太低或者不需要再训练了；如果比例太高，说明学习率可能太高。

确保每层的激活参数及梯度分布

如果每层权重呈现全是 0 或在 -1 和 1 上，那肯定是出问题了。

第一层特征可视化

左图特征充满了噪音，网络可以出问题，没有收敛，学习率不恰当，正则化的惩罚权重过低等。右图的特征不错，平滑，干净且种类繁多。

其他超参数

学习率衰减率，更新类型，正则化，网络结构（隐藏层数量、深度等）

理解正则化，在SVM/Softmax的例子中，正则化损失从生物学角度可以看做逐渐遗忘，因为它的效果是让所有突触权重在参数更新过程中逐渐向着0变化。

其他问题

如果 loss 在开始时很平滑，之后突然开始训练，可能是参数初始化不好，刚开始没学到东西，在某个点突然的变化。

属性分类

当一个样本上可能有多个标签时，对样本 $i$ 而言，为每个属性 $j$ 创建独立的二分类器，

当样本 i 包括属性 j 时，$y{ij} = 1$，$1-y{ij}f_j = 1-f_j$，当 $f_j$ 得分小于 1 时，会产生损失值；

当样本 i 不包括属性 j 时，$y{ij} = -1$，$1-y{ij}f_j = 1+f_j$，当 $f_j$ 得分大于 -1 时，会产生损失值。

回归问题

回归问题中通常计算预测值和真实值之间的损失。采用 L2 范式或 L1 范式

如果是多个属性，用 $\Sigma_j$ 将所有属性加起来。对于 $f_j$ 求导，

求导后，评分值的梯度要么与误差中的差值直接成比例，要么是固定的并从差值中继承sign。

在实践中，L2 损失优化比起 Softmax 损失，优化要困难得多。因为网络需要一个评分的正确值，而且在鲁棒性也不是很好，异常值可以导致很大的梯度。但是 Softmax 损失每个评分的准确值不那么重要，分类也能给出关于回归的分布。所以在面对回归任务时，尽量把输出变成二分类，变成一个分类问题。

面对实际的回归问题时，先看能否将输出二值化，转化成多个二分类问题。比如对一个产品的星级进行预测时，使用5个独立的分类器对 1-5 星打分比回归损失要好得多。
