sqlalchemy 踩坑集锦

发表于 2019-10-10 | 分类于 Bug ， sql |
| 字数统计: 1k

记录 sqlalchemy 使用中遇到的一些问题

全局 Session 导致的 connection go away

在更改 SQLAlchemy Session 从每次请求都创建到一个全局 Session 之后遇到了如下问题：

StatementError: (sqlalchemy.exc.InvalidRequestError) Can’t reconnect until invalid transaction is rolled back [SQL: ]

或者是

raised unexpected: OperationalError(“(_mysql_exceptions.OperationalError) (2006, ‘MySQL server has gone away’)”,)

错误是 SQLAlchemy 抛出。原因是你从 pool 拿的 connection 没有以 session.commit 或 session.rollback 或者 session.close 放回 pool 里。这时 connection 的 transaction 没有完结（rollback or commit)。 而不知什么原因（recyle 了，timeout 了）你的 connection 又死掉了，你的 sqlalchemy 尝试重新连接。由于 transaction 还没完结，无法重连。

正确用法是确保 session 在使用完成后用 session.close, session.commit 或者 session.rollback 把连接还回 pool。

主要进行以下几个方面设置：

  1. 增加装饰器函数，确保在每次请求，访问操作数据库后，transaction 能够终止，而不是一直开放。 后面发现这种方法不是最优的，仍然没有解决问题。

  2. 利用 @main.before_request 在请求开始时打开 session，利用 @main.teardown_request 在请求结束时关闭 session

  1. 设置 pool_recycle 时间

    app.config['SQLALCHEMY_POOL_RECYCLE'] = 3600

  2. 通过 db.create_scoped_session() 保证线程安全

在利用 session 进行 update 或 add 后，查询不到记录的问题

flask-sqlalchemy 查询结果有两种形式，

一种是利用 Table.query.filter() 进行查询，

一种利用 session.query(Table).filter 进行查询。

在利用 session.update() 或者 session.add() 进行更新时，如果在 session.flush() 后仍使用第一种查询方法，常常会出现一些问题

  1. 在执行 session.add() 后，使用 Table.query.filter() 没有查询到插入的变异，发生以下报错

    sqlalchemy.exc.IntegrityError: (pymysql.err.IntegrityError) (1062, "Duplicate entry 'cn_del-exon12-13-2501' for key 'cnv_unique_idx'")

  2. 在更新后，执行 session.flush() 时，出现以下错误，启用的两次不同的 session

    sqlalchemy.exc.InvalidRequestError: Object '<Sample at 0x7f38809d32e8>' is already attached to session '1' (this is '2')

所以，在执行查询时，尽量保证全局使用的同一个 session，出现 sqlalchemy 报错时，首先应该想到的是，是不是 session 不统一导致的。

数据库中某个字段的值不在 sqlalchemy Table 定义的枚举值里

在 query 之后出现一下报错

SQLAlchemy LookupError: “DIVERSE” is not among the defined enum values

这时需要修改数据库中的值或者修改 Table 中枚举类型字段值

在 SQLALCHEMY_BINDS 绑定不同数据库后，查询时仍然出现数据库中表不存在的情况

例如，表 Student 和表 Salary 分别属于不同数据库 dbA 和 dbB，使用 SQLALCHEMYBINDS 进行绑定，并且指定 _bind_key 后，将两个表按以下方式 join 后

会出现 dbA.Salary 不存在的 Bug。

原因是虽然进行了 _bindkey 的绑定，但在查询时，由于表 Student 在前，整条查询语句默认的数据库是 Student 所在的 dbA，所以，找不到 dbB 中的 Salary，会出现 dbA.Salary 不存在的 Bug。

解决思路有三个

一是，属于不同数据库的表分别进行查询，最后将结果合并，而不使用 join 来连接两张表的结果

二是，在 sql 中使用 join，使用 session.execute(sql) 得到查询结果

三是，通过设定 schema 指定不同的数据库表，参考以下两个连接 https://stackoverflow.com/questions/44564369/join-tables-in-two-databases-using-sqlalchemy 和 https://stackoverflow.com/questions/11438352/many-to-many-multi-database-join-with-flask-sqlalchemy?rq=1

第三种方法是比较麻烦的。
